# 特典0: Codex初期セットアップ一括プロンプト

このファイルの下にあるプロンプトを、Codex にそのまま貼ってください。

Codex が、推奨フォルダ構成と最初に使う md ファイルをまとめて作成します。すでに同名のフォルダやファイルがある場合は、上書きせずに確認してから進めるように指示しています。

## 使い方

1. Codex で、これから作業したい親フォルダを開く
2. 下のプロンプトを丸ごと貼る
3. Codex が提示する作成予定を確認する
4. 問題なければ作成してもらう

## コピペ用プロンプト

```text
あなたは、私のCodex作業環境を初期セットアップするアシスタントです。

目的:
Codexを購入直後から迷わず使えるように、推奨フォルダ構成と最初に使うMarkdownファイルを作成してください。

重要ルール:
- まず現在の作業フォルダを確認してください。
- 既存ファイルや既存フォルダを上書きしないでください。
- 同名ファイルがある場合は、上書きせず、作成を止めて確認してください。
- 削除や移動はしないでください。
- 最初は最小構成で作ってください。凝った構成にしすぎないでください。
- 作成後に、次に私がやることを3つだけ教えてください。

作成してほしいフォルダ:
Codex_Workspace/
├── 00_inbox/
├── 01_sources/
├── 02_notes/
├── 03_prompts/
├── 04_outputs/
├── 05_assets/
└── 99_archive/

作成してほしいファイル:

1. Codex_Workspace/AGENTS.md
内容:
## Daily Work Agent

- まず依頼のゴールを1文で確認してください
- 不明点が多い場合は、推測で広げすぎず、前提を明示してください
- 出力はまずMarkdownで保存しやすい形を優先してください
- 長い説明より、先に実行案と次の一歩を示してください
- 既存ファイルを変更する前に、変更対象と理由を短く説明してください
- 削除や上書きが必要な場合は、先に確認してください

2. Codex_Workspace/02_notes/START_HERE.md
内容:
# START HERE

このフォルダはCodex作業用の場所です。

最初にやること:
1. `01_sources/` に参考資料を3つ入れる
2. `03_prompts/` によく使う依頼文を保存する
3. `04_outputs/` にCodexの成果物を出す

ルール:
- 元ファイルと成果物を混ぜない
- 迷ったファイルは `00_inbox/` に入れる
- 古いものは削除せず、まず `99_archive/` へ移す

3. Codex_Workspace/03_prompts/FIRST_PROMPTS.md
内容:
# FIRST PROMPTS

## 1. 作業整理

この作業をCodexに依頼するとしたら、ゴール、参照ファイル、出力形式、制約条件、確認観点の5箱で整理してください。

## 2. 要約

この文章を3点で要約してください。最後に「次に読むべき人」を1行で添えてください。

## 3. AGENTS.md整理

この作業でAGENTS.mdに入れるべき恒久ルールと、今回限りのプロンプトを分けてください。

4. Codex_Workspace/04_outputs/README.md
内容:
# Outputs

ここにはCodexに作ってもらった成果物を保存します。

注意:
- 原本はここに置かない
- 途中版と完成版が混ざる場合は、日付や用途がわかる名前にする
- 重要な成果物は、あとで見返せるように短い説明を添える

完了後に出してほしい内容:
- 作成したフォルダ一覧
- 作成したファイル一覧
- 次に私がやること3つ
```

## 作成後にやること

- `01_sources/` に、過去記事、過去投稿、考え方メモなどを3つだけ入れる
- `AGENTS.md` を見て、自分に合わない行があれば1行だけ直す
- `03_prompts/FIRST_PROMPTS.md` から1本選んで、Codexに試す
