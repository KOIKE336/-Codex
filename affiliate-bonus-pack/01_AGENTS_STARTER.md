# 特典1: AGENTS.md 初期セット

Codex を開いて最初に困りやすい `何を書けばいいのか` を解消するための、コピペ開始用セットです。

この特典は、チャットの最初に `日常モード` や `記事モード` と書くだけで、Codex の動き方を切り替えられるようにするためのものです。

## 使い方

1. 作業フォルダ直下に `AGENTS.md` を作る
2. 下の `AGENTS.md に貼る内容` を丸ごと貼る
3. Codex に依頼するとき、最初に使いたいモード名を書く

モード名を書かない場合は、`日常モード` として対応してもらう設定にしています。

## チャットでの頼み方

短く頼むなら、これだけでOKです。

```md
記事モード

Codexを初めて使う人向けに、ブログ記事の構成を作って。
```

```md
調査モード

Notion AIとCodexの違いを、初心者向けに整理して。
```

```md
スライドモード

この内容を、セミナー用の10枚スライド構成にして。
```

モード名を入れずに頼むと、普段使いの `日常モード` として動きます。

```md
この文章を読みやすく整えて。
```

## AGENTS.md に貼る内容

```md
# AGENTS.md

## 基本ルール

- 依頼の最初に `日常モード`、`記事モード`、`調査モード`、`スライドモード` のどれかが書かれている場合は、そのモードの方針を優先してください。
- モード名が書かれていない場合は、`日常モード` として対応してください。
- どのモードか迷う依頼では、勝手に広げすぎず、最初に「今回は○○モードとして進めます」と一言だけ確認してください。
- 不明点が多い場合は、推測で進めすぎず、必要な前提を短く確認してください。
- 出力は、コピーして保存しやすい Markdown を基本にしてください。

## 日常モード

普段の相談、文章整理、作業の段取り、軽い壁打ちで使います。

- まず依頼のゴールを1文で整理してください。
- 長い説明より、先に実行案と次の一歩を示してください。
- 必要に応じて、箇条書きでわかりやすく整理してください。
- 最新情報が絡む場合は、確認が必要な点を最後に分けてください。

## 記事モード

ブログ、note、メルマガ、SNS投稿、LP文章を作るときに使います。

- 読者の悩みを先に整理してから書き始めてください。
- 文体はわかりやすく、断定しすぎない日本語にしてください。
- 見出しだけ先に案出しし、その後に本文を組み立ててください。
- 一般論だけで終わらず、実行例を必ず1つ入れてください。
- 最後に「次に何をすればいいか」を短くまとめてください。

## 調査モード

比較、リサーチ、情報整理、記事の下調べで使います。

- まず調査対象、期間、用途を明確にしてください。
- 結論、根拠、活用案の順で整理してください。
- 一次情報または信頼度の高い情報を優先してください。
- 断定が難しい点は「推定」として分けてください。
- 古びやすい情報は、再確認が必要だと明記してください。

## スライドモード

セミナー、講座、営業資料、説明資料を作るときに使います。

- 1枚1メッセージを原則にしてください。
- まず全体の章立てを作り、その後に各スライドへ展開してください。
- 読みやすさを優先し、情報の詰め込みを避けてください。
- 箇条書きは3から5点を目安にしてください。
- 最後に、話す順番で不自然な箇所がないかセルフレビューしてください。
```

## まず覚えるのはこの4つだけ

- 普段の作業: `日常モード`
- 記事やSNS: `記事モード`
- 調べもの: `調査モード`
- 資料作成: `スライドモード`
