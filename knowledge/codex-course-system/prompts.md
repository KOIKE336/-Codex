# Prompt Templates

## 1. 章ごとの取り込み

```text
この教材チャンクを読んで、次の4区分で整理してください。

1. durable
今後も腐りにくい原理・運用・考え方

2. volatile
時期依存の主張、UI、モデル名、価格、機能一覧

3. executable
AGENTS.md / Skill / Automation / Checklist / Prompt に変換できるもの

4. revalidate
公式情報で再確認すべき主張

出力条件:
- 箇条書き中心
- source_id を残す
- 最後に「この章をどう資産化するか」を3行でまとめる
```

## 2. 動画講義の資産化

```text
この動画の文字起こしを、教材としてではなく知識資産として再構成してください。

必ず次を出してください。
- 要点5つ
- 使えるプロンプト案3つ
- AGENTS.md に移すべき恒久ルール
- Skill 化できる作業手順
- 陳腐化しやすい説明

前提:
- 動画の言い回しは残しすぎず、再利用しやすい文章へ圧縮する
- UI案内、価格、モデル推奨は volatile 扱いにする
```

## 3. 教材全体の評価

```text
この教材全体を evaluation-rubric.md に沿って評価してください。

見たい観点:
- 初心者教材としての強さ
- 実運用ノウハウとしての強さ
- 他ツールへ移植できる知識の多さ
- 古びやすい箇所
- 自分専用ルールに落とし込める箇所

出力条件:
- スコア表を含める
- Keep / Rewrite / Revalidate を分ける
- 最後に「この教材から先に資産化すべき3つ」を書く
```

## 4. AGENTS.md 抽出

```text
この教材チャンクから、毎回のプロンプトではなく AGENTS.md に常駐させるべきルールだけを抽出してください。

抽出基準:
- 何度も繰り返し使う
- 案件固有ではなく運用固有
- 判断基準や禁止事項として効く

出力形式:
## Candidate Rules
- ...

## Not AGENTS.md
- 今回限りなのでプロンプトに残すべきもの
```

## 5. Skill 候補抽出

```text
この教材の中から Skill にしやすい作業を抽出してください。

各候補について:
- Skill名
- 入力
- 手順
- 出力
- 向いている場面
- 向いていない場面

出力は、再利用優先で具体的にしてください。
```
