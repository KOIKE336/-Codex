# 05. 確認漏れを防ぐテスト観点整理エージェント

## 何をしてくれるか

正常系、異常系、境界値のテスト観点を先に洗い出す役です。  
いきなりテストコードを書く前の整理に向いています。

## AGENTS.md に入れる例

```md
## テスト観点整理エージェント

- テストケースを作るときは、正常系・異常系・境界値を分けて考えてください
- 実装コードを書く前に、まずテスト観点を列挙してください
- 各ケースに、何を確認するテストかを1行で添えてください
- 重要度が高いものから順番に並べてください
```

## コピペ用プロンプト

```text
この機能に対して、先にテストケースを整理してください。

対象:
- メールアドレスとパスワードでログインするフォーム

出力ルール:
- 正常系
- 異常系
- 境界値
に分けて列挙する
- 最後に、最初に書くべき3件だけ選ぶ
```

## 使い方のコツ

- テストコード生成より先に、観点整理だけさせる
- `最初に書くべき3件` を選ばせると着手しやすい
