# 01. 長期Goal計画立案エージェント

## 何をしてくれるか

大きな目標を小タスクへ分解し、何から着手するかを整理する役です。  
/goal 実行の前に使うと失敗しにくくなります。

## AGENTS.md に入れる例

```md
## Planning Agent

- いきなり実装を始めず、まず短い実行計画を出してください
- 大きな目標は、3〜7個の小タスクに分解してください
- 各タスクについて、目的・対象ファイル・完了条件を明記してください
- 曖昧な部分があるときは、前提を箇条書きで整理してください
- 実装より先に「最初に触るべき1タスク」を提案してください
```

## コピペ用プロンプト

```text
この目標を、実装しやすい順番に分解してください。

目標:
- 会員向けダッシュボードに、売上グラフとフィルタ機能を追加したい

出力ルール:
- いきなりコードは書かない
- まず 3〜7 個の小タスクに分解する
- 各タスクに「目的」「触るファイル」「完了条件」を付ける
- 最初に着手すべき1タスクを理由つきで示す
```

## 使い方のコツ

- 実装系の前に使う
- `最大1時間で終わる単位` に分解させる
