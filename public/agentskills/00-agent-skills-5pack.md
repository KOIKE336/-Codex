# Codexで使い回せるAGENT SKILLS 5選

このファイルは、5本の Agent Skill をまとめて読める全体版です。

## 収録

1. 長期Goal計画立案エージェント
2. 確認漏れを防ぐテスト観点整理エージェント
3. 人に見せる前の差分チェックエージェント
4. 不具合の再現と原因切り分けエージェント
5. 見た目の仕上げチェックエージェント

## 使い方

1. まず 1 本だけ選ぶ
2. `AGENTS.md に入れる例` をプロジェクト直下の `AGENTS.md` に入れる
3. 作業時に `コピペ用プロンプト` を送る

役割分担:

- `AGENTS.md` = 毎回守ってほしい進め方
- `今回のプロンプト` = 今日やってほしい依頼

似て見えるスキルもありますが、役割より「使うタイミング」で分けると迷いにくいです。

- 着手前: 長期Goal計画立案
- 実装直後: テスト観点整理
- 提出前: 差分チェック
- 不具合発生時: バグ切り分け
- 仕上げ前: UI仕上げ

## 1. 長期Goal計画立案エージェント

### 役割

大きな目標を小タスクへ分解し、何から着手するかを整理する役です。

### AGENTS.md に入れる例

```md
## Planning Agent

- いきなり実装を始めず、まず短い実行計画を出してください
- 大きな目標は、3〜7個の小タスクに分解してください
- 各タスクについて、目的・対象ファイル・完了条件を明記してください
- 実装より先に「最初に触るべき1タスク」を提案してください
```

### コピペ用プロンプト

```text
この目標を、実装しやすい順番に分解してください。

目標:
- 会員向けダッシュボードに、売上グラフとフィルタ機能を追加したい

出力ルール:
- いきなりコードは書かない
- まず 3〜7 個の小タスクに分解する
- 各タスクに「目的」「触るファイル」「完了条件」を付ける
```

## 2. 確認漏れを防ぐテスト観点整理エージェント

### 役割

正常系、異常系、境界値の確認観点を先に洗い出す役です。

### AGENTS.md に入れる例

```md
## テスト観点整理エージェント

- テストケースを作るときは、正常系・異常系・境界値を分けて考えてください
- 実装コードを書く前に、まず確認観点を列挙してください
- 重要度が高いものから順番に並べてください
```

### コピペ用プロンプト

```text
この機能に対して、先に確認観点を整理してください。

対象:
- メールアドレスとパスワードでログインするフォーム

出力ルール:
- 正常系
- 異常系
- 境界値
に分けて列挙する
- 最後に、最初に確認すべき3件だけ選ぶ
```

## 3. 人に見せる前の差分チェックエージェント

### 役割

変更差分を先に読み、明らかなバグ・テスト漏れ・危ない変更を洗い出す役です。

### AGENTS.md に入れる例

```md
## 差分チェックエージェント

- 差分レビューでは、まず明らかなバグ・想定外入力への対応漏れ・テスト不足を確認してください
- 指摘は「必ず直したい」「余裕があれば改善したい」に分けてください
- 結果は短く、箇条書きで返してください
```

### コピペ用プロンプト

```text
この変更差分を、Pull Request を出す前のレビューとして確認してください。

見てほしい観点:
- 明らかなバグ
- 想定外の入力や分岐への対応漏れ
- テストが足りない箇所
- セキュリティ上の気になる点
```

## 4. 不具合の再現と原因切り分けエージェント

### 役割

バグ報告やログを受け取って、再現手順・原因候補・最初に見るべき場所を整理する役です。

### AGENTS.md に入れる例

```md
## バグ切り分けエージェント

- バグ調査では、まず再現手順を明文化してください
- ログ・画面・関連コードから、原因候補を2〜3個に絞ってください
- 結果は「再現手順」「怪しい場所」「次の確認」の3つで返してください
```

### コピペ用プロンプト

```text
このバグ報告とログをもとに、原因を切り分けてください。

バグ報告:
- ログイン後に一覧画面が真っ白になる

ログ:
- console に "Cannot read properties of undefined" が出る
```

## 5. 見た目の仕上げチェックエージェント

### 役割

画面を見て、情報量、余白、強弱、AIっぽさの有無をレビューする役です。

### AGENTS.md に入れる例

```md
## UIチェック&改善エージェント

- UIを確認するときは、まず情報密度と視線誘導を見てください
- 余白、見出しの強弱、配色、カードのメリハリを具体的に指摘してください
- 改善案は優先順位つきで返してください
```

### コピペ用プロンプト

```text
このUIを確認して、初心者向けの特典ページとして読みやすいかレビューしてください。

見てほしい観点:
- 情報量が詰まりすぎていないか
- 見出しと本文の強弱があるか
- 余白が足りているか
```
