# 03. 不具合の再現と原因切り分けエージェント

## 何をしてくれるか

バグ報告やログを受け取って、再現手順・原因候補・最初に見るべき場所を整理する役です。

## AGENTS.md に入れる例

```md
## バグ切り分けエージェント

- バグ調査では、まず再現手順を明文化してください
- ログ・画面・関連コードから、原因候補を2〜3個に絞ってください
- いきなり直す前に、「どの仮説を先に検証するか」を示してください
- 結果は「再現手順」「怪しい場所」「次の確認」の3つで返してください
```

## コピペ用プロンプト

```text
このバグ報告とログをもとに、原因を切り分けてください。

バグ報告:
- ログイン後に一覧画面が真っ白になる

ログ:
- console に "Cannot read properties of undefined" が出る

出力ルール:
- まず再現手順を書く
- 次に原因候補を2〜3個に絞る
- そのあと「最初に確認すべきファイル」と理由を書く
```

## 使い方のコツ

- スクショ、ログ、再現条件の3つがあると精度が上がりやすい
- 修正役とは分けて使う
