# 02. 人に見せる前の差分チェックエージェント

## 何をしてくれるか

変更差分を先に読み、明らかなバグ・テスト漏れ・危ない変更を洗い出す役です。  
人間レビューの前に `下ごしらえ` として使います。

## AGENTS.md に入れる例

```md
## 差分チェックエージェント

- 差分レビューでは、まず明らかなバグ・想定外入力への対応漏れ・テスト不足を確認してください
- 人間が見るべき重い設計判断と、先に直せる軽い修正を分けてください
- 指摘は「必ず直したい」「余裕があれば改善したい」に分けてください
- 結果は短く、箇条書きで返してください
```

## コピペ用プロンプト

```text
この変更差分を、Pull Request を出す前のレビューとして確認してください。

見てほしい観点:
- 明らかなバグ
- 想定外の入力や分岐への対応漏れ
- テストが足りない箇所
- セキュリティ上の気になる点

最後に、レビュー担当へ見せる短い要約も書いてください。
```

## 使い方のコツ

- 最初は3〜4観点に絞る
- 人間レビューの代わりではなく、事前整形として使う
