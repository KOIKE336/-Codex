# 特典2: Codex用フォルダ構成テンプレ

最初に作業範囲をきれいに分けておくと、事故が減って精度も上がります。

## 推奨構成

```text
Codex_Workspace/
├── 00_inbox/
├── 01_sources/
├── 02_notes/
├── 03_prompts/
├── 04_outputs/
├── 05_assets/
└── 99_archive/
```

## 各フォルダの役割

- `00_inbox`
  - とりあえず入れる場所
- `01_sources`
  - 参考資料、過去記事、文字起こし
- `02_notes`
  - 考え方メモ、構成案、要件メモ
- `03_prompts`
  - よく使う依頼文テンプレ
- `04_outputs`
  - Codex に作らせた成果物
- `05_assets`
  - AGENTS.md、Skill案、チェックリスト
- `99_archive`
  - 終了済み、古い版、退避用

## 最初に入れると良い素材

- 過去のブログ記事3本
- 過去のSNS投稿20本
- 自分の考え方メモ1本
- 参考にしたいスライド1本
- 音声や動画の文字起こし

## ルール

- 動画や音声は、そのままより文字起こしを優先
- 大事なファイルは `04_outputs` に直接上書きさせない
- 削除より `99_archive` へ移動を優先
- まずは1フォルダ1テーマに絞る

